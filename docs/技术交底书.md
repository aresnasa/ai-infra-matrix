交底书撰写注意事项：
1.软件类发明不需要给出代码，用自然语言从逻辑上描述技术方案；
2.如果创造点涉及网络架构的变化，则需要分别给出相关技术的网络架构图以及本发明的网络架构图；如果网络架构没有变化，创造点涉及数据处理流程的变化，则需要分别给出相关技术的数据处理流程以及本发明的数据处理流程；
3.介绍数据流程时，请按照数据流的走向、时间顺序进行描述；
4.专利法规定予以保护的是技术方案不是功能，因此需要给出实现某种功能的具体技术方案，不能仅仅阐述功能；
5．为了获得一个较大的保护范围，在能够实现最基本的发明目的的前提条件下，发明人应该在当前实施的最优技术方案基础上发散思维，给出尽可能多的替代方案；
6．代理人不是技术专家，当代理人电话咨询时发明人需要耐心讲解；如果代理人要求补充必要的资料，发明人应当及时补充；
7.交底书撰写完毕后，请将蓝色字体的内容删除。

2. 本发明的技术方案(核心内容，需详细描述)

 2-2. 替代方案
